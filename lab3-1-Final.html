Representancin de Tokens o Texto

Como bien hemos hablado varias veces, la computadora no entiende palabras ni mucho menos oraciones completas en la misma forma que nuestros cerebros lo hacen. Por ello, debemos encontrar alguna forma de representar palabras o caracteres en una manera que la computadora sea capaz de interpretarla, es decir, con nmeros. Hay varias formas de representar un grupo de palabras de forma numrica, pero para fines de este laboratorio vamos a centrarnos en una manera comn, llamada "one-hot encoding".

One Hot Encoding

Esta tcnica debe resultarles familiar de cursos pasados, donde se tomaba una conjunto de categoras y se les asignaba una columna por categora, entonces se coloca un 1 si el row que estamos evaluando es parte de esa categora o un 0 en caso contrario. Este mismo acercamiento podemos tomarlo para representar conjuntos de palabras. Por ejemplo

casa = [1, 0, 0, ..., 0]
perro = [0, 1, 0, ..., 0]

Representar un vocabulario grande con one-hot enconding, suele volverse ineficiente debido al tamao de cada vector disperso. Para solventar esto, una prctica comn es truncar el vocabulario para contener las palabras ms utilizadas y representar el resto con un smbolo especial, UNK, para definir palabras "desconocidas" o "sin importancia". A menudo esto se hace que palabras tales como nombres se vean como UNK porque son raros.

def seqs_to_dicts(sequences):
    """
    Crea word_to_idx y idx_to_word para una lista de secuencias
    
    Args:
    sequences: lista de secuencias a usar
    
    Returns:
    Diccionario de palabra a indice
    Diccionario de indice a palabra
    Int numero de secuencias
    Int tamao del vocabulario
    """
    
    # Lambda para aplanar (flatten) una lista de listas
    flatten = lambda l: [item for sublist in l for item in sublist]
    
    # Aplanamos el dataset
    all_words = flatten(sequences)
    
    # Conteo de las ocurrencias de las palabras
    word_count = defaultdict(int)
    for word in all_words:
        word_count[word] += 1
        
    # Ordenar por frecuencia
    word_count = sorted(list(word_count.items()), key=lambda x: -x[1])
    
    # Crear una lista de todas las palabras nicas
    unique_words = [w[0] for w in word_count]
    
    # Agregamos UNK a la lista de palabras
    unique_words.append("UNK")
    
    # Conteo del nmero de secuencias y el nmero de palabras unicas
    num_sentences, vocab_size = len(sequences), len(unique_words)
    
    # Crear diccionarios mencionados
    word_to_idx = defaultdict(lambda: vocab_size-1)
    idx_to_word = defaultdict(lambda: 'UNK')
    
    # Llenado de diccionarios
    for idx, word in enumerate(unique_words):
        word_to_idx[word] = idx
        idx_to_word[idx] = word
        
    return word_to_idx, idx_to_word, num_sentences, vocab_size

class Dataset(data.Dataset):
    def __init__(self, inputs, targets):
        self.inputs = inputs
        self.targets = targets

    def __len__(self):
        # Return the size of the dataset
        return len(self.targets)

    def __getitem__(self, index):
        # Retrieve inputs and targets at the given index
        X = self.inputs[index]
        y = self.targets[index]

        return X, y

    
def create_datasets(sequences, dataset_class, p_train=0.8, p_val=0.1, p_test=0.1):
    
    # Definimos el tamao de las particiones
    num_train = int(len(sequences)*p_train)
    num_val = int(len(sequences)*p_val)
    num_test = int(len(sequences)*p_test)

    # Dividir las secuencias en las particiones
    sequences_train = sequences[:num_train]
    sequences_val = sequences[num_train:num_train+num_val]
    sequences_test = sequences[-num_test:]

    # Funcion interna para obtener los targets de una secuencia
    def get_inputs_targets_from_sequences(sequences):
        # Listas vacias
        inputs, targets = [], []
        
        # Agregar informacion a las listas, ambas listas tienen L-1 palabras de una secuencia de largo L
        # pero los targetes estn movidos a la derecha por uno, para que podamos predecir la siguiente palabra
        for sequence in sequences:
            inputs.append(sequence[:-1])
            targets.append(sequence[1:])
            
        return inputs, targets

    # Obtener inputs y targes para cada subgrupo
    inputs_train, targets_train = get_inputs_targets_from_sequences(sequences_train)
    inputs_val, targets_val = get_inputs_targets_from_sequences(sequences_val)
    inputs_test, targets_test = get_inputs_targets_from_sequences(sequences_test)

    # Creacin de datasets
    training_set = dataset_class(inputs_train, targets_train)
    validation_set = dataset_class(inputs_val, targets_val)
    test_set = dataset_class(inputs_test, targets_test)

    return training_set, validation_set, test_set

Redes Neuronales Recurrentes (RNN)

Una red neuronal recurrente (RNN) es una red neuronal conocida por modelar de manera efectiva datos secuenciales como el lenguaje, el habla y las secuencias de protenas. Procesa datos de manera cclica, aplicando los mismos clculos a cada elemento de una secuencia. Este enfoque cclico permite que la red utilice clculos anteriores como una forma de memoria, lo que ayuda a hacer predicciones para clculos futuros. Para comprender mejor este concepto, consideren la siguiente imagen.

Crdito de imagen al autor, imagen tomada de "Introduction to Recurrent Neural Network" de Aishwarya.27

Donde:

  • $x$ es la secuencia de input
  • $U$ es una matriz de pesos aplicada a una muestra de input dada
  • $V$ es una matriz de pesos usada para la computacin recurrente para pasar la memroia en las secuencias
  • $W$ es una matriz de pesos usada para calcular la salida de cada paso
  • $h$ es el estado oculto (hidden state) (memoria de la red) para cada paso
  • $L$ es la salida resultante

Cuando una red es extendida como se muestra, es ms facil referirse a un paso $t$. Tenemos los siguientes calculos en la red

  • $h_t=f(U x_t + V h_{t-1}$ donde f es la funcin de activacion
  • $L_t = softmax(W h_t)$

def init_orthogonal(param):
    """
    Initializes weight parameters orthogonally.
    Inicializa los pesos ortogonalmente
    
    Esta inicializacin est dada por el siguiente paper:
    https://arxiv.org/abs/1312.6120
    """
    if param.ndim < 2:
        raise ValueError("Only parameters with 2 or more dimensions are supported.")

    rows, cols = param.shape
    
    new_param = np.random.randn(rows, cols)
    
    if rows < cols:
        new_param = new_param.T
    
    # Calcular factorizacin QR
    q, r = np.linalg.qr(new_param)
    
    # Hacer Q uniforme de acuerdo a https://arxiv.org/pdf/math-ph/0609050.pdf
    d = np.diag(r, 0)
    ph = np.sign(d)
    q *= ph

    if rows < cols:
        q = q.T
    
    new_param = q
    
    return new_param


def init_rnn(hidden_size, vocab_size):
    """
    Inicializa la RNN
    
    Args:
     hidden_size:  Dimensiones del hidden state
     vocab_size: Dimensin del vocabulario
    """
    # 1. Inicializacin de matrices con las formas correctas
    # U: input -> hidden
    U = np.zeros((hidden_size, vocab_size))
    
    # V: hidden (prev) -> hidden (next)
    V = np.zeros((hidden_size, hidden_size))
    
    # W: hidden -> output
    W = np.zeros((vocab_size, hidden_size))
    
    # Bias del hidden state
    b_hidden = np.zeros((hidden_size, 1))
    
    # Bias de salida
    b_out = np.zeros((vocab_size, 1))

    # 2. Inicializacin ortogonal de las matrices de pesos
    U = init_orthogonal(U)
    V = init_orthogonal(V)
    W = init_orthogonal(W)

    # 3. Devolver los parmetros como una tupla
    return U, V, W, b_hidden, b_out

def sigmoid(x, derivative=False):
    """
    Calcula la funcin sigmoide para un array x

    Args:
     x: El array sobre el que trabajar
     derivative: Si esta como verdadero, regresar el valor en la derivada
    """
    x_safe = x + 1e-12  # Evitar ceros
    f = 1 / (1 + np.exp(-x_safe))
    
    if derivative: 
        return f * (1 - f)
    else: 
        return f


def tanh(x, derivative=False):
    """
    Calcula la funcin tanh para un array x

    Args:
     x: El array sobre el que trabajar
     derivative: Si esta como verdadero, regresar el valor en la derivada
    """
    x_safe = x + 1e-12  # Evitar ceros
    f = np.tanh(x_safe)
    
    if derivative: 
        return 1 - f ** 2
    else: 
        return f


def softmax(x, derivative=False):
    """
    Calcula la funcin softmax para un array x

    Args:
     x: El array sobre el que trabajar
     derivative: Si esta como verdadero, regresar el valor en la derivada
    """
    x_safe = x - np.max(x)  # Para estabilidad numrica
    exp_x = np.exp(x_safe)
    f = exp_x / np.sum(exp_x, axis=0, keepdims=True)
    
    if derivative: 
        pass  # No se necesita explcitamente para backpropagation con cross-entropy
    else: 
        return f

def forward_pass(inputs, hidden_state, params):
    """
    Calcula el paso forward de RNN
    
    Args:
     inputs: Secuencia de input a ser procesada (lista de vectores one-hot o ndices)
     hidden_state: Un estado oculto inicial (array de shape [hidden_size, 1])
     params: Parmetros de la RNN
    """
    # Obtener los parmetros
    U, V, W, b_hidden, b_out = params
    
    # Crear listas para guardar las salidas y los hidden states
    outputs, hidden_states = [], []
    
    # Para cada elemento en la secuencia input
    for t in range(len(inputs)):
        x_t = inputs[t].reshape(-1, 1)  # Asegura que sea columna

        # 1. Calcular el hidden state
        hidden_state = tanh(np.dot(U, x_t) + np.dot(V, hidden_state) + b_hidden)

        # 2. Calcular la salida (distribucin de probabilidad)
        out = softmax(np.dot(W, hidden_state) + b_out)

        # Guardar resultados
        outputs.append(out)
        hidden_states.append(hidden_state.copy())
    
    return outputs, hidden_states

    """ 
    # Setea el mximo de la norma para que sea flotante
    max_norm = float(max_norm)
    total_norm = 0
    
    # Calculamos la norma L2 al cuadrado para cada gradiente y agregamos estas a la norma total
    for grad in grads:
        grad_norm = np.sum(np.power(grad, 2))
        total_norm += grad_norm
    # Cuadrado de la normal total
    total_norm = np.sqrt(total_norm)
    
    # Calculamos el coeficiente de recorte
    clip_coef = max_norm / (total_norm + 1e-6)
    
    # Si el total de la norma es ms grande que el mximo permitido, se recorta la gradiente
    if clip_coef < 1:
        for grad in grads:
            grad *= clip_coef
    return grads


def backward_pass(inputs, outputs, hidden_states, targets, params):
    """
    Calcula el paso backward de la RNN
    
    Args:
     inputs: secuencia de input
     outputs: secuencia de output del forward
     hidden_states: secuencia de los hidden_state del forward
     targets: secuencia target
     params: parametros de la RNN
    """
    
    # Obtener los parametros
    U, V, W, b_hidden, b_out = params
    
    # Inicializamos las gradientes como cero (Noten que lo hacemos para los pesos y bias)
    d_U, d_V, d_W = np.zeros_like(U), np.zeros_like(V), np.zeros_like(W)
    d_b_hidden, d_b_out = np.zeros_like(b_hidden), np.zeros_like(b_out)
    
    # Llevar el record de las derivadas de los hidden state y las perdidas (loss)
    d_h_next = np.zeros_like(hidden_states[0])
    loss = 0
    
    # Iteramos para cada elemento en la secuencia output
    # NB: Iteramos de regreso sobre t=N hasta 0
    for t in reversed(range(len(outputs))):

        loss += -np.sum(targets[t] * np.log(outputs[t] + 1e-12)) / len(targets[t])        
        
        d_o = outputs[t].copy()
        d_o[np.argmax(targets[t])] -= 1
        
        d_W += np.dot(d_o, hidden_states[t].T)

        d_b_out += d_o
        
        d_h = np.dot(W.T, d_o) + d_h_next
        
        d_f = d_h * tanh(hidden_states[t], derivative=True)

        d_b_hidden += d_f
        
        d_U += np.dot(d_f, inputs[t].T)
        
        d_V  += np.dot(d_f, hidden_states[t-1].T)

        d_h_next = np.dot(V.T, d_f)
    
    # Empaquetar las gradientes
    grads = d_U, d_V, d_W, d_b_hidden, d_b_out    
    
    # Corte de gradientes
    grads = clip_gradient_norm(grads)
    
    return loss, grads

def update_parameters(params, grads, lr=1e-3):
    # Iteramos sobre los parametros y las gradientes
    for param, grad in zip(params, grads):
        param -= lr * grad

Preguntas

Ya hemos visto el funcionamiento general de nuestra red RNN, viendo las grficas de arriba, responda lo siguiente dentro de esta celda

  • Qu interpretacin le da a la separacin de las graficas de training y validation?

    Una separacin creciente entre las curvas de entrenamiento y validacin indica que el modelo est sobreajustando (overfitting). Esto significa que est memorizando las secuencias del conjunto de entrenamiento en lugar de generalizar a datos no vistos. Si la prdida de validacin deja de mejorar o comienza a aumentar mientras la de entrenamiento sigue disminuyendo, ese es un signo claro de sobreajuste.

  • Cree que es un buen modelo basado solamente en el loss?

    Evaluar un modelo solo con la funcin de prdida puede ser insuficiente. El loss puede parecer bajo, pero si no se acompaa de una buena capacidad de generalizacin (por ejemplo, validado con accuracy u otra mtrica relevante en validacin/testing), no necesariamente indica buen desempeo. Adems, un loss bajo en entrenamiento pero alto en validacin sugiere sobreajuste.

  • Cmo deberan de verse esas grficas en un modelo ideal?

    En un modelo ideal, ambas curvas (training y validation) deberan disminuir juntas de forma suave. Puede haber una ligera separacin, pero ambas deberan seguir una tendencia descendente sin que la validacin se dispare. Esto indicara que el modelo est aprendiendo correctamente sin memorizar ni sobreajustar.

Crditos: La segunda parte de este laboratorio est tomado y basado en uno de los laboratorios dados dentro del curso de "Deep Learning" de Jes Frellsen (DeepLearningDTU)

Consideren leer el siguiente blog para mejorar el entendimiento de este tema: http://colah.github.io/posts/2015-08-Understanding-LSTMs/

La RNN estndar enfrenta un problema de gradientes que desaparecen, lo que dificulta la retencin de memoria en secuencias ms largas. Para hacer frente a estos desafos, se introdujeron algunas variantes.

Los dos tipos principales son la celda de memoria a corto plazo (LSTM) y la unidad recurrente cerrada (GRU), las cuales demuestran una capacidad mejorada para conservar y utilizar la memoria en pasos de tiempo posteriores.

En este ejercicio, nuestro enfoque estar en LSTM, pero los principios aprendidos aqu tambin se pueden aplicar fcilmente para implementar GRU.

Recordemos una de las imagenes que vimos en clase

Crdito de imagen al autor, imagen tomada de "Designing neural network based decoders for surface codes" de Savvas Varsamopoulos

Recordemos que la "celula" de LST contiene tres tipos de gates, input, forget y output gate. La salida de una unidad LSTM est calculada por las siguientes funciones, donde $\sigma = softmax$. Entonces tenemos la input gate $i$, la forget gate $f$ y la output gate $o$

  • $i = \sigma ( W^i [h_{t-1}, x_t])$
  • $f = \sigma ( W^f [h_{t-1},x_t])$
  • $o = \sigma ( W^o [h_{t-1},x_t])$

Donde $W^i, W^f, W^o$ son las matrices de pesos aplicada a cada aplicadas a una matriz contatenada $h_{t-1}$ (hidden state vector) y $x_t$ (input vector) para cada respectiva gate $h_{t-1}$, del paso previo junto con el input actual $x_t$ son usados para calcular una memoria candidata $g$

  • $g = tanh( W^g [h_{t-1}, x_t])$

El valor de la memoria $c_t$ es actualizada como

$c_t = c_{t-1} \circ f + g \circ i$

donde $c_{t-1}$ es la memoria previa, y $\circ$ es una multiplicacion element-wise (recuerden que este tipo de multiplicacin en numpy es con *)

La salida $h_t$ es calculada como

$h_t = tanh(c_t) \circ o$

y este se usa para tanto la salida del paso como para el siguiente paso, mientras $c_t$ es exclusivamente enviado al siguiente paso. Esto hace $c_t$ una memoria feature, y no es usado directamente para caluclar la salida del paso actual.

def forward(inputs, h_prev, C_prev, p):
    """
    Arguments:
    inputs: Lista de vectores one-hot del input (uno por paso de tiempo), cada uno shape (vocab_size, 1)
    h_prev: Hidden state en el paso "t-1", shape (hidden_size, 1)
    C_prev: Memoria en el paso "t-1", shape (hidden_size, 1)
    p: Lista con pesos y biases
    Returns:
    z_s, f_s, i_s, g_s, C_s, o_s, h_s, v_s, output_s: Listas con los clculos de cada paso forward
    """
    
    # Validar las dimensiones
    assert h_prev.shape == (hidden_size, 1)
    assert C_prev.shape == (hidden_size, 1)

    # Desempacar los parametros
    W_f, W_i, W_g, W_o, W_v, b_f, b_i, b_g, b_o, b_v = p
    
    # Listas para calculos de cada componente en LSTM
    x_s, z_s, f_s, i_s  = [], [], [], []
    g_s, C_s, o_s, h_s  = [], [], [], []
    v_s, output_s       = [], []

    # Agregar los valores iniciales 
    h_s.append(h_prev)
    C_s.append(C_prev)
    
    for x in inputs:

        # 1. Concatenar x y h_prev  z
        z = np.vstack((h_prev, x))
        z_s.append(z)
        
        # 2. Forget gate
        f = sigmoid(np.dot(W_f, z) + b_f)
        f_s.append(f)
        
        # 3. Input gate
        i = sigmoid(np.dot(W_i, z) + b_i)
        i_s.append(i)
        
        # 4. Memoria candidata
        g = tanh(np.dot(W_g, z) + b_g)
        g_s.append(g)
        
        # 5. Actualizar memoria (c_t)
        C_prev = f * C_prev + i * g
        C_s.append(C_prev)
        
        # 6. Output gate
        o = sigmoid(np.dot(W_o, z) + b_o)
        o_s.append(o)
        
        # 7. Calcular nuevo hidden state
        h_prev = o * tanh(C_prev)
        h_s.append(h_prev)

        # 8. Logits
        v = np.dot(W_v, h_prev) + b_v
        v_s.append(v)
        
        # 9. Output con softmax
        output = softmax(v)
        output_s.append(output)

    return z_s, f_s, i_s, g_s, C_s, o_s, h_s, v_s, output_s

    o: Calculos del output gate como una lista de tamao m.

    v: Calculos del logit como una lista de tamao m.

                        W_f:  Pesos de la forget gate, shape (n_a, n_a + n_x)

                        W_i: Pesos de la update gate, shape (n_a, n_a + n_x)
                        b_i: Bias de la update gate, shape (n_a, 1)

Responda lo siguiente dentro de esta celda

  • Qu modelo funcion mejor? RNN tradicional o el basado en LSTM? Por qu?

    El modelo basado en LSTM funcion mejor. Esto se debe a que las LSTM estn diseadas especficamente para lidiar con el problema del gradiente desvanecido que afecta a las RNN tradicionales, permitiendo que se mantenga la informacin relevante durante ms pasos de tiempo. En la prctica, observamos que el modelo LSTM converge ms rpido y con menor prdida tanto en entrenamiento como validacin.

  • Observen la grfica obtenida arriba, en qu es diferente a la obtenida a RNN? Es esto mejor o peor? Por qu?

    En la grfica de LSTM, la prdida de validacin y entrenamiento tienden a seguir trayectorias ms estables y cercanas, con menos separacin que en el caso de la RNN. Esto es mejor, ya que indica que el modelo generaliza mejor y no est sobreajustando tan fuertemente. Adems, las oscilaciones suelen ser menores, lo que refleja un aprendizaje ms consistente.

  • Por qu LSTM puede funcionar mejor con secuencias largas?

    LSTM introduce una memoria explcita que se actualiza controladamente mediante gates (input, forget y output), lo que le permite aprender a retener o descartar informacin en cada paso. Esto hace que el modelo pueda propagar informacin relevante a lo largo de secuencias largas sin que el gradiente se degrade rpidamente, lo cual es un gran problema en RNN tradicionales.

        self.lstm = nn.LSTM(input_size=vocab_size,
                            hidden_size=50,
                            num_layers=1,
                            bidirectional=False,
                            batch_first=False)

        self.l_out = nn.Linear(in_features=50,
                               out_features=vocab_size,

        x, (h, c) = self.lstm(x)

        x = x.view(-1, self.lstm.hidden_size)
        
        # Output layer (sin softmax, porque se usa cross-entropy)

El tamao de la ventana, denotado por w, dicta cuntos datos puede considerar el modelo al hacer la prediccin. Este parmetro tambin se conoce como look back period (perodo retrospectivo).

Entonces, creemos una funcin para obtener estos datos, dado un look back period. Adems, debemos asegurarnos de transformar estos datos a tensores para poder ser usados con PyTorch.

Esta funcin est diseada para crear ventanas en la serie de tiempo mientras predice un paso de tiempo en el futuro inmediato. Su propsito es convertir una serie de tiempo en un tensor con dimensiones (muestras de ventana, pasos de tiempo, caractersticas). Dada una serie de tiempo con t pasos de tiempo, puede producir aproximadamente (t - ventana + 1) ventanas, donde "ventana" denota el tamao de cada ventana. Estas ventanas pueden comenzar desde cualquier paso de tiempo dentro de la serie de tiempo, siempre que no se extiendan ms all de sus lmites.

Cada ventana contiene mltiples pasos de tiempo consecutivos con sus valores correspondientes, y cada paso de tiempo puede tener mltiples caractersticas. Sin embargo, en este conjunto de datos especfico, solo hay una funcin disponible.

La eleccin del diseo garantiza que tanto la "caracterstica" como el "objetivo" tengan la misma forma. Por ejemplo, para una ventana de tres pasos de tiempo, la "caracterstica" corresponde a la serie de tiempo de t-3 a t-1, y el "objetivo" cubre los pasos de tiempo de t-2 a t. Aunque estamos principalmente interesados en predecir t+1, la informacin de t-2 a t es valiosa durante el entrenamiento.

import torch

def create_timeseries_dataset(dataset, lookback):
    X, y = [], []
    for i in range(len(dataset) - lookback):
        feature = dataset[i : i + lookback]
        target = dataset[i + 1 : i + lookback + 1]
        X.append(feature)
        y.append(target)
    return torch.tensor(X), torch.tensor(y)

class CustomModelLSTM(nn.Module):
    def __init__(self, input_size=1, hidden_size=100, num_layers=1, output_size=1):
        super(CustomModelLSTM, self).__init__()
        self.hidden_size = hidden_size
        self.num_layers = num_layers

        self.lstm = nn.LSTM(input_size, hidden_size, num_layers)

        self.fc = nn.Linear(hidden_size, output_size)

    def forward(self, x):
        h0 = torch.zeros(self.num_layers, x.size(1), self.hidden_size)
        c0 = torch.zeros(self.num_layers, x.size(1), self.hidden_size)

        out, _ = self.lstm(x, (h0, c0))

        out = self.fc(out)

        # Pasar por capa final
        return out

La funcin nn.LSTM() produce una tupla como salida. El primer elemento de esta tupla consiste en los hidden states generados, donde cada paso de tiempo de la entrada tiene su correspondiente hidden state. El segundo elemento contiene la memoria y los hidden states de la unidad LSTM, pero no se usan en este contexto particular.

La capa LSTM se configura con la opcin batch_first=True porque los tensores de entrada se preparan en la dimensin de (muestra de ventana, pasos de tiempo, caractersticas). Con esta configuracin, se crea un batch tomando muestras a lo largo de la primera dimensin.

Para generar un nico resultado de regresin, la salida de los estados ocultos se procesa an ms utilizando una capa fully connected. Dado que la salida de LSTM corresponde a un valor para cada paso de tiempo de entrada, se debe seleccionar solo la salida del ltimo paso de tiempo.
